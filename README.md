# Adaprive Learning Navigator

## О проекте

Adaprive Learning Navigator — это веб-сервис для изучения IT-направлений в формате roadmap-first.

Текущий вектор проекта:
- пользователь выбирает готовое IT-направление из каталога roadmap'ов;
- сервис показывает структуру тем, зависимости и материалы;
- на основе лимита часов в неделю и уже известных тем строится личный недельный план;
- AI используется не как основной генератор roadmap, а как будущий помощник по конкретной теме.

По сути, проект движется в сторону аналога `roadmap.sh`, но:
- с русскоязычным интерфейсом;
- с личным недельным темпом обучения;
- с учётом уже знакомых тем;
- с перспективой AI Tutor внутри темы.

## Что уже реализовано

### Пользовательский функционал

- регистрация, вход и обновление JWT-токенов;
- защищённая приватная часть приложения;
- многостраничный frontend внутри Spring Boot;
- каталог roadmap'ов;
- просмотр roadmap по выбранному направлению;
- просмотр деталей темы, её зависимостей и ресурсов;
- построение личного плана по roadmap на основе `hoursPerWeek`;
- просмотр сохранённых планов;
- страница недельного плана;
- базовые страницы `Profile` и `Progress` как часть общей структуры продукта;
- legacy-режим AI-генерации плана всё ещё сохранён в backend, но это уже не основной сценарий продукта.

### Реализованные разделы интерфейса

#### Главная `/`
- кратко объясняет, что это за сервис;
- показывает основную пользу для нового пользователя;
- ведёт к регистрации или входу.

#### Вход `/login`
- авторизация по email и паролю;
- после успеха переводит пользователя в кабинет.

#### Регистрация `/register`
- создание аккаунта;
- после успешной регистрации пользователь получает токены и доступ к приватной части.

#### Кабинет `/dashboard`
- стартовая точка после входа;
- показывает каталог roadmap'ов;
- позволяет выбрать направление;
- позволяет указать часов в неделю;
- позволяет отметить уже известные темы;
- строит и сохраняет личный план по выбранному roadmap.

#### Roadmap `/roadmap`
- показывает структуру выбранного направления;
- отображает темы, связи, зависимости и ресурсы;
- даёт панель деталей по выбранной теме;
- содержит заготовку под будущий AI Tutor.

#### План `/plan`
- показывает сохранённый личный план по неделям;
- отображает часы, темы недели и объяснение порядка тем.

#### Профиль `/profile`
- содержит пользовательские настройки и заготовки под персональные предпочтения.

#### Прогресс `/progress`
- подготовлен как экран для отслеживания прохождения тем;
- сейчас служит основой для следующего этапа интеграции серверного progress.

## Текущая продуктовая модель

Сейчас в проекте существуют два сценария:

### 1. Основной сценарий
Roadmap-first:
- выбрать roadmap;
- просмотреть структуру тем;
- исключить уже известные темы;
- построить план по неделям на основе количества часов.

### 2. Legacy-сценарий
AI-generated route:
- запросить генерацию плана через AI;
- backend валидирует результат и сохраняет план.

Этот сценарий пока оставлен в проекте, но больше не является основной логикой интерфейса.

## Техническая реализация

### Backend

Стек:
- Java 17;
- Spring Boot 4;
- Spring Web MVC;
- Spring Security;
- Spring Data JPA;
- H2 Database;
- Spring Validation;
- Spring AI + Ollama;
- JWT authentication.

Ключевые backend-модули:
- `AuthController` — регистрация, логин, refresh;
- `PlanController` — получение и построение планов;
- `RoadmapController` — каталог roadmap'ов и детали тем;
- `PageController` — маршрутизация HTML-страниц;
- `AuthServiceImpl` — auth-логика;
- `PlanServiceImpl` — построение и сохранение персонального плана;
- `RoadmapServiceImpl` — read-model для roadmap'ов;
- security-слой на JWT;
- global exception handling.

### Frontend

Текущий frontend построен как MPA внутри Spring Boot:
- отдельные HTML-страницы;
- общий CSS;
- общий JS для auth, session и API;
- без отдельного SPA framework на текущем этапе.

Основные frontend-ресурсы:
- `src/main/resources/static/index.html`
- `src/main/resources/static/login.html`
- `src/main/resources/static/register.html`
- `src/main/resources/static/dashboard.html`
- `src/main/resources/static/roadmap.html`
- `src/main/resources/static/plan.html`
- `src/main/resources/static/profile.html`
- `src/main/resources/static/progress.html`

Общие frontend-модули:
- `assets/js/core/api.js` — вызовы API;
- `assets/js/core/session.js` — хранение токенов, выбранных roadmap/plan и клиентского состояния;
- `assets/js/core/ui.js` — форматирование и вспомогательный UI-слой;
- `assets/css/app.css` — общая дизайн-система и стили страниц.

### Конфигурация

Основные настройки находятся в:
- `src/main/resources/application.yaml`

Что важно:
- приложение работает на `http://localhost:8080`;
- H2 используется как файловая база;
- SQL init для текущей dev-базы выключен (`spring.sql.init.mode=never`);
- браузер может открываться автоматически при старте;
- для AI используется Ollama и модель `qwen2.5:7b`.

### Запуск

Варианты:
- запуск из IDE;
- запуск через Maven.

После старта приложение открывает сайт на главной странице автоматически, если это разрешено окружением и настройками.

## AI и Ollama

В проекте уже есть интеграция со Spring AI и Ollama.

На текущем этапе:
- AI-маршрутизация всё ещё доступна как legacy-функция;
- основной новый вектор — использовать AI как topic-scoped Tutor.

Это означает:
- AI должна перейти из роли "строит весь маршрут" в роль "помогает разбирать конкретную тему".
